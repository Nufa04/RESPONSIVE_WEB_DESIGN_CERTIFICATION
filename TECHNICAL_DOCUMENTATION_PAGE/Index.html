Introduction to MyDigital ID

MyDigital ID is the official national digital identity platform developed for the citizens and residents of Malaysia. It serves as a secure, centralized, and highly sophisticated authentication ecosystem that eliminates the need for multiple usernames and passwords across various digital platforms.

By establishing a single trusted identity verifier, this platform empowers individuals to seamlessly log into both federal government portals and participating private sector applications. The infrastructure is highly scalable, designed to support millions of concurrent transactions while ensuring data privacy is maintained at the highest national standards.

As Malaysia accelerates its digital economy transformation, MyDigital ID plays a pivotal role in fostering a secure online environment, reducing identity theft, and streamlining digital interactions for businesses, government agencies, and the general public alike.

What you should already know

Before diving into the technical integration of MyDigital ID within your enterprise software or web services, developers are highly encouraged to possess a solid baseline understanding of modern web architectures and cryptographic protocols.

Having prior experience in these core areas will drastically reduce your development timeline and help ensure your implementation adheres to strict security guardrails:

  • A strong grasp of the OpenID Connect (OIDC) framework and OAuth 2.0 authorization flows.
  • Familiarity with token-based authentication concepts, specifically JSON Web Tokens (JWT).
  • Experience in managing server-side secure environment variables, API secrets, and private-public key pairs.
  • Basic knowledge of web application security hazards, including cross-site scripting (XSS) and token interception methods.
Prerequisites for Developers

Before integrating the national identity system into your application, ensure you meet the following baseline requirements:

Here are the core security parameters and credentials required by the Malaysian digital gateway:

  • An active Malaysian Government Developer Center (MGD) merchant account.
  • A valid SSL/TLS certificate installed on your production server.
  • A unique API Client ID and Secret Key generated via the administrator dashboard.

SDK Installation Methods

To speed up development, the government core team maintains modern software development kits (SDKs).

You can install the official package directly into your web framework using standard package managers:

npm install @mydigitalid/core-sdk
User Authentication Flow

The system utilizes a secure OpenID Connect protocol to verify the identity of the user via biometric scanners or mobile pins.

To initiate the secure login challenge, your web application must execute the following primary initiation function:

MyDigitalID.initializeSecureChallenge({ clientId: "MY_APP_01" });
Response Data Structure

Upon successful authentication, the Malaysian central server will return an encrypted token payload back to your server.

The decrypted payload will strictly provide verified demographic information associated with the national database.

var userPayload = { country: "Malaysia", citizenStatus: true };
API Endpoint Verification

To ensure that your network traffic is routing directly to the authentic national gateway, developers must bind their production environment requests to the designated verification address.

Always perform a standard ping test or diagnostic request to the production URI string during the staging deployment phase:

const productionGatewayURI = "https://api.mydigitalid.gov.my/v1/verify";
